@property

  • @property后面可以跟哪些修饰符？

    • 线程安全的角度，主要包括nonatomic和atomic
    • 控制访问权限的角度，主要包括readonly和readwrite
    • 内存管理的角度，strong，weak，assign，copy，retain
    • 自定义setter和getter：setter和getter

  • 什么情况使用 weak 关键字，相比 assign 有什么不同？

    • weak的主要作用是在ARC下解决循环引用的问题，weak不需要手动设置为nil，而assgin需要手动设置为nil，不然就会造成野指针，另外assgin可以修饰基本类型，而weak只能修饰OC对象

  • 怎么用 copy 关键字？

    • copy关键字主要是本身指向一个不可变的对象副本，这样在原有对象修改的时候不会影响自身的改变。

  • 如何让自己的类用 copy 修饰符？

    • 支持NSCopying或者NSMutableCopying协议，实现该协议中的copyWithZone方法。

  • @property的本质是什么，ivar，getter和setter如何生成并添加到类中？

    • @property包括了ivar和存取方法，ivar和getter和setter是编译器通过@synthesize关键字指定自动合成的，编译器已经默认为我们添加了@synthesize，所以不需要显式指定@synthesize关键字。

  • @protocol 和 category 中如何使用 @property？

    • protocol和category中都只会生成getter和setter的方法声明，在protocol中支持该协议的对象需要实现该属性，而在category中则采用关联对象的方式增加属性。

  • runtime 如何实现 weak 属性

    • runtime会将weak对象放入hash表中，然后以weak对象指向的地址作为key，当对象调用dealloc的时候，通过key寻找到weak对象，将其设置为nil。

  • weak需要在dealloc中设置为nil么

    • weak不需要在dealloc中设置为nil，当weak对象释放的时候，系统会自动将其标记为nil。

  • ARC下，不显示指定任何关键字，默认的关键字有哪些？

    • atomic， readwrite， strong， assgin

  • 用@property声明的NSString（或NSArray，NSDictionary）经常使用copy关键字，为什么？如果改用strong关键字，可能造成什么问题？

    • 用copy相当于重新开辟了一块内存空间，新对象指向新开辟的内存空间，原对象值更改并不会对新对象造成影响，strong仍旧指向原有的对象地址，一旦原有的对象值发生改变，新对象的值也同样发生改变。

  • @synthesize合成实例变量的规则是什么？假如property名为foo，存在一个名为_foo的实例变量，那么还会自动合成新变量么？

    • 如果指定了成员变量的名称，则会生成一个指定名称的成员变量
    • 如果成员变量已经存在，则不会生成
    • 如果没有指定成员变量的名称，会自动生成一个与属性名同名的成员变量

  • 在有了自动合成属性实例变量之后，@synthesize还有哪些使用场景？

    • 自定义getter，setter
    • readonly自定义getter
    • @dynamic
    • protocol中定义的属性
    • category中声明的属性（利用关联对象）
    • 重写属性

runtime

  • objc中向一个nil对象发送消息将会发生什么？

    • 什么也不发生。objc在向一个对象发送消息时，runtime库会根据对象的isa指针找到该对象实际所属的类，然后在该类中的方法列表以及其父类方法列表中寻找方法运行，然后在发送消息的时候，objc_msgSend方法不会返回值，所谓的返回内容都是具体调用时执行的。那么，如果向一个nil对象发送消息，首先在寻找对象的isa指针时就是0地址返回了，所以不会出现任何错误。

  • objc中向一个对象发送消息[obj foo]和objc_msgSend()函数之间有什么关系？

    • 当向对象发送消息的时候，runtime 会将[obj foo]转成objc_msgSend(obj, foo)，之后去obj对应的类中去寻找foo方法，首先先从cache中寻找，如果不存在则去methodList中寻找，如果仍然不存在去其父类的methodList中去寻找，如果仍然不存在查看其是否实现了resolveInstanceMethod或者resolveClassMethod，如果仍旧没有则执行forwardingTargetForSelector，最后执行forwardInvocation。

  • 什么时候会报unrecognized selector的异常？

    • 参考上题，通过runtime去查找方法实现，如果没有方法实现尝试着去做resolve，如果还没有的话进行方法转发，如果仍旧没有实现转发，那么就会报出该异常。

  • 一个objc对象如何进行内存布局？（考虑有父类的情况）

    • 经典的图，实例对象的isa指向其类对象，类对象的isa指向元类，元类的isa指向根元类，根元类的isa指向自身，类对象的super指向其父类，一直到NSObject，NSObject的super指向nil，元类的super也指向其父类，一直到根元类，根元类的super指向NSObject。

  • 一个objc对象的isa的指针指向什么？有什么作用？

    • 一个objc对象的isa指针指向该对象所属的类，类的isa指向元类，元类的isa指向元类的根类，根元类的isa指针指向自身，isa可以帮助对象找到该方法。

  • runtime如何通过selector找到对应的IMP地址？（分别考虑类方法和实例方法）

    • 在OC的内存分布中，每一个类对象都有方法列表，里面存在着方法名称和方法实现等，实例方法从类对象的方法列表里寻找，而类方法从元类对象的方法列表里寻找，当然前提都是先查缓存。

  • 使用runtime Associate方法关联的对象，需要在主对象dealloc的时候释放么？

    • 使用关联对象的方式，在主对象进行dealloc的时候不需要手动进行释放。

  • objc中的类方法和实例方法有什么本质区别和联系？

    • 类方法主要针对类对象，其内部不可以调用实例方法，实例方法主要针对实例对象，其可以通过类名调用类方法。

  • _objc_msgForward函数是做什么的，直接调用它将会发生什么？

    • 做方法转发的时候，runtime会调用_objc_msgForward，直接调用将会直接触发转发操作。

  • runtime如何实现weak变量的自动置nil？

    • runtime将会以weak变量的地址作为key，将weak对象放入一个hash表中，当对象引用计数为0的时候就会以该变量地址为key，在hash表中以该key为键搜索对象，将其设置为nil。

  • 能否向编译后得到的类中增加实例变量？能否向运行时创建的类中添加实例变量？为什么？

    • 不可以向编译后的类中增加实例变量，可以向运行时创建的类中增加实例变量，编译后的类，其内存布局已经确定，运行时创建的类中，可以添加变量。

runloop

  • runloop和线程有什么关系？

    • 每一个线程都有一个runloop，runloop的主要作用就是用来管理线程的，有了runloop，线程执行完任务后，就可以一直处于休眠等待状态，而不用退出。这样做的好处就是不需要反复创建线程，减少创建开销。

  • runloop的mode作用是什么？

    • mode公开的Mode有两种，NSDefaultRunLoopMode和NSRunLoopCommonMode，主要是设定事件执行的优先级。

  • 以+ scheduledTimerWithTimeInterval…的方式触发的timer，在滑动页面上的列表时，timer会暂定回调，为什么？如何解决？

    • 在调用该方法触发timer的时候，timer被加载NSDefaultRunLoopMode上，当列表滑动的时候，runloop被切换到UITrackingRunLoopMode来确保列表滑动的流畅性，因此会导致timer不会被调用，因此我们可以将timer添加到NSRunLoopCommonMode来解决。

  • 猜想RunLoop内部是如何实现的？

    • 内部应该就是一个do-while循环，里面不断处理各种事件，每个线程和RunLoop对象唯一对应，另外每个RunLoop在启动的时候只能指定其中一个Mode，每个Mode中包含Source，Timer以及Observer，主线程的RunLoop默认启动，其他需要手动启动。

内存管理

  • objc使用什么机制管理对象内存？

    • objc主要使用引用计数的方式管理对象的内存。

  • ARC通过什么方式帮助开发者管理内存？

    • ARC使开发者不用关心内存管理，它主要采用自动引用计数的方式，自动为开发者添加release，autoRelease等

  • 不手动指定autoreleasepool的前提下，一个autorealese对象在什么时刻释放？（比如在一个vc的viewDidLoad中创建）

    • 所有的autorelease对象在出了作用域会被添加到最近的autoreleasepool中，之后当前runloop结束时候会进行释放。

  • BAD_ACCESS在什么情况下出现？

    • 当前对象已被释放，程序仍去访问

  • 苹果是如何实现autoreleasepool的？

    • 每一个线程的 autoreleasepool 其实就是一个指针的堆栈
    • 每一个指针代表一个需要 release 的对象或者 POOL_SENTINEL（哨兵对象，代表一个 autoreleasepool 的边界）
    • 这个堆栈被划分成了一个以 page 为结点的双向链表。pages 会在必要的时候动态地增加或删除
    • Thread-local storage（线程局部存储）指向 hot page ，即最新添加的 autoreleased 对象所在的那个 page

Block

  • 使用block时什么情况会发生引用循环，如何解决？

    • 当对象强引用了block，block内部又强引用了该对象，通过weak或者block修饰符来解决。

  • 在block内如何修改block外部变量？

    • 使用__block来继续修饰变量，使得该变量为block所持有。

  • 使用系统的某些block api（如UIView的block版本写动画时），是否也考虑引用循环问题？

    • 系统的block api基本无需考虑循环引用的问题，但额外的GCD引用self，而其他参数均为自身属性，仍然需要考虑循环引用。

GCD

  • GCD的队列（dispatch_queue_t）分哪两种类型？

    • GCD的队列主要分为单队列Serial Queue和并发队列ConCurrent Queue。

  • 如何用GCD同步若干个异步调用？

    • 可以通过GCD group来同步若干个异步调用。

  • dispatch_barrier_async的作用是什么？

    • dispatch_barrier_async主要的应用场景主要为，等待一些任务执行完成之后，再执行barrier追加的处理，之后该并发队列才会恢复之前的操作，目的是为了避免数据竞争。

  • 苹果为什么要废弃dispatch_get_current_queue？

    • 这个应该是在iOS7之后就废弃了，废弃他最主要的原因就是容易造成死锁。

KVO & KVC

  • addObserver:forKeyPath:options:context:各个参数的作用分别是什么，observer中需要实现哪个方法才能获得KVO回调？

    • 观察者，观察的属性，观察的选项，上下文，observer需要实现- (void)observeValueForKeyPath:(NSString )keyPath ofObject:(id)object change:(NSDictionary )change context:(void *)context;才能获取KVO的回调。

  • 如何手动触发一个value的KVO

    • 手动触发需要用到NSObject的方法：willChangeValueForKey，和didChangeValueForKey。

  • 若一个类有实例变量NSString *_foo，调用setValue:forKey:时，可以以foo还是_foo作为key？

    • 都可以

  • KVC的keyPath中的集合运算符如何使用？

    • 类似valueForKeyPath:@”sum.a”

  • KVC和KVO的keyPath一定是属性么？

    • 不一定

  • 如何关闭默认的KVO的默认实现，并进入自定义的KVO实现？

    • 实现类方法, +(BOOL)automaticallyNotifiesObserversForKey:(NSString *)key, 根据需要返回NO, 然后实现
      -(void)willChangeValueForKey:(NSString )key和
      -(void)didChangeValueForKey:(NSString )key

  • apple用什么方式实现对一个对象的KVO？

    • runtime创建了一个新类继承自原来的类，之后重写观察属性的setter方法，最后将这个对象的isa指针指向新创建的子类实例。

  • 如何调试BAD_ACCESS错误

    • 全局断点
    • 僵尸对象
    • 重写object的respondsToSelector方法，现实出现EXEC_BAD_ACCESS前访问的最后一个object

  • lldb（gdb）常用的调试命令？

    • po
    • bt
    • print
    • br
    • p
    • expr
