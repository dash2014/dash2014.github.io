微信小程序之粗见

发表于 2017-11-24 | 分类于 tech

大概半个多月前，微信公众平台推送了小程序支持了新的开放能力支持web-view，即小程序支持内嵌网页了，可以把你开发的h5网页直接拿过来使用了，使用的方法也非常简单，只需要在小程序管理配置后台增加支持的业务域名，之后在你的wxml页面上增加web-view标签，将url地址配置给src即可，其默认支持全屏展开，那开放了内嵌网页的能力到底意味着什么？说一说我对小程序的粗见吧。茶余饭后，简单想想，并不深入，记录感触而已。

为什么要开发小程序

小程序发布伊始，无数公司趋之若鹜，使用小程序的原因很多，随便列举几个：

  • 微信强大的资源入口，微信的顶级流量众所周知，一众开发者、商家都想通过微信的品牌为自己的产品打call，以此来实现自己的流量增长。

  • 微信技术号召力，在国内的一众互联网公司中，BAT的技术积累以及号召力无出其右，加上张小龙亲自背书，一时间小程序风生水起。

  • 开发者需要技术储备，任何一个公司都会对新技术做一定量的调研和技术储备，以此来应对即将可能到来的变化。

  • write once，run anywhere，小程序出来的时机正是移动端跨平台风风火火的一段时间，大家都打着write once，run anywhere的标语，而小程序甚至也可看作一次跨平台尝试，特别是其类h5的写法，当然它是基于微信平台，这就为很多个人开发者以及初创公司打开了一扇门。

  • 上手简单，针对独立开发者和业务方向不明朗的初创型公司，需要实现一个相对简单的功能，不需要额外的移动端开发者，节省开发成本，可以反复试错。

  • 及时更新上线，这条就应该是针对苹果的审核了，苹果的审核规则复杂，审核周期略长，虽然已有所改进。但小程序不需要经过苹果审核，自然也免去了这个周期。

等等。。。

为什么不想使用小程序

  • 微信入口隐藏，很明显张小龙并不想因为小程序而影响对于微信的正常使用，然而又需要利用微信的流量优势，故而造了一个势，吸引各路开发者前来，却将小程序的入口隐藏。

  • 开发者之殇，优秀的小程序并没有被大力推广，甚至你仍然需要搜索才能够找到，这除了给微信增加了很多流量之外，自己并没有得到太多的优势。

  • 复杂业务，小程序开放能力不足，并不足以应对复杂的业务流，公司发展到一定程度，其业务复杂性可想而知，小程序并不足以支撑这种业务的扩张。

  • 公司的品牌，任何一个公司都需要有自己的核心产品，并坚持为它打call，比如美团，比如点评，微信，支付宝等等，这个核心产品才是撑起公司战略和公司发展的基石，小程序应该是锦上添花之举。

  • 使用效果，大家也知道，小程序并不如native的使用效果更好，即使其开发周期短，但当今的用户在追求优秀的产品idea同时，也更加注重产品性能，使用便捷等因素，或者简单的称之为手感。

等等。。。

小程序的未来

小程序的未来在哪？虽然我对张小龙，及微信团队怀着很大的敬仰，但我仍旧认为其失败了，特别是web-view的开放，这真的是对h5的一种妥协。

  • 其类h5的语法虽说与h5的开发很神似，但仍旧需要额外的工作来适配，这不得不说是一种麻烦。

  • 适配了h5，我为什么不选择直接使用h5开发做分享传播？而选择入口极深的小程序嵌入h5？

  • 虽说倡导开源代码，但开源的更多是工具代码，真心把业务代码交付给另一个平台，这无论如何都不如公司自己持有代码来的更符合逻辑吧。

可能还有很多更深层次的原因吧，饭后闲想，对于我个人来说，简单的尝试写一写，了解一下其实现的做法，实现个简单的小业务，仅此。而如果小程序后续真的出现了颠覆性的革命性的事件，再继续跟进吧。

native终将变的更好，我也相信web终将也会变的更好。
