runtime好文推荐

发表于 2017-07-02 | 分类于 tech

最近项目比较忙，忙于工程重构和改造，忙于工程细节的补充和整理，疏于笔书，之前一直在看runtime相关源码，一直想动手写一篇关于runtime的文章，直到我看到了以下几篇runtime相关文章，差点有了弃笔从戎的赶脚，下面就简单的推荐一下这几篇，有一些runtime源码的基础，有一些是从一些不错的角度进行切入来分析runtime的设计，不再赘述。

  • 第一篇，神经病院Objective-C Runtime入院第一天——isa和Class, 这篇文章从几个问题出发，着手分析了isa与Class的相关，包括对isa内部的变量介绍都比较详细，之后对这几个问题通过源码进行了分析，还是比较值得一看的。

  • 第二篇，神经病院Objective-C Runtime住院第二天——消息发送与转发,这篇文章介绍了消息的发送与转发的基本流程，包括方法的查找流程，cache的使用，resolveMethod，以及forwardMethod，包括对源码进行了详细的介绍和拆解，最后给出了整体流程的绘制图，大家可以参考一下。

  • 第三篇，神经病院Objective-C Runtime出院第三天——如何正确使用Runtime，这篇文章从多个角度介绍了runtime的用法，个人觉得非常不错，总结下来大概这样几个用法：

    • 利用runtime实现多继承

    • 方法替换

    • 切面变成

    • ISA替换

    • 关联对象

    • 简单的方法增加

    • 自动归档和解档

    • 字典模型互转

      参考这篇文章，就可以整体的把iOS中runtime的用法大概了解清楚，推荐一下。

  • 第四篇，从 NSObject 的初始化了解 isa,通过NSObject初始化了解isa，与前面的文章略有重合，不过这算是读取runtime源码的一个切入角度，走一遍初始化，与下一篇文章的套路接近。

  • 第五篇，Objc 对象的今生今世,这篇文章从对象的创建，初始化，使用，销毁几个角度进行了runtime的源码分析，非常不错

  • 第六篇，深入解析 ObjC 中方法的结构，这篇文章介绍了Objc如何存储方法，包括给出的结论，

    • 类在内存中的位置是在编译期间决定的，在之后修改代码，也不会改变内存中的位置。

    • 类的方法、属性以及协议在编译期间存放到了“错误”的位置，直到 realizeClass 执行之后，才放到了 class_rw_t 指向的只读区域 class_ro_t，这样我们即可以在运行时为 class_rw_t 添加方法，也不会影响类的只读结构。

    • 在 class_ro_t 中的属性在运行期间就不能改变了，再添加方法时，会修改 class_rw_t 中的 methods 列表，而不是 class_ro_t 中的 baseMethods，对于方法的添加会在之后的文章中分析。

以上，参考上述文章，对照runtime源码，便可很快的了解到OC对象在内存中的定义，以及OC对象的方法调用等流程，如此不错的文章，个人就不再狗尾续貂了，大家参考就是~~~
