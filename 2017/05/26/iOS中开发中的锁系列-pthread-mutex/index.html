iOS中开发中的锁系列-pthread_mutex

发表于 2017-05-26 | 分类于 tech

在NSLock里面了解过，NSLock只不过是简单的pthread_mutex的OC封装，于是便有了了解一下pthread_mutex的想法。

pthread是POSIX的标准库，很多操作系统都有自己的相应实现，mutex通常可以理解为互斥锁，意思呢就代表锁保护的临界区只能允许一个线程进入，其他未获得锁权限的线程，只能等待。

关于pthread_mutex_lock的定义，可以参考这里，在这篇描述里，可以看到mutex定义了四种mutex type，分别是NORMAL，ERRORCHECK，RECURSIVE，DEFAULT，其中NORMAL和DEFAULT，不做任何检测，不建议使用。所以通常采用ERRORCHECK，它会做错误检测，如果尝试着对已上锁的mutex继续上锁，或者尝试解锁未上锁的mutex，会导致错误。RECURSIVE就是递归锁，之后再有一篇来专门总结一下。

在iOS中使用pthread_mutex的方式如下

  • 初始化： pthread_mutex_init(&theTaskMutex, NULL);

  • 加锁： pthread_mutex_lock(&theTaskMutex);

  • 解锁： pthread_mutex_unlock(&theTaskMutex);

  • 释放：pthread_mutex_destroy(&theTaskMutex);

关于pthread_mutex的底层实现，具体可以看一下这篇文章。另外查找了半天的pthread_mutex的实现代码，可以参考一下这篇文章。

由于pthread_mutex是较底层的锁，所以其效率会比OC实现的锁高一些，YYKit里面之前由于自旋锁的问题，而将所有内部需要用到锁的地方，都改成了pthread_mutex。
