iOS中开发中的锁系列-NSLock

发表于 2017-05-25 | 分类于 tech

NSLock比较简单，所以本篇幅应该也会比较简短，查了一些资料，苹果的官方文档是这样描述NSLock的。

The NSLock class uses POSIX threads to implement its locking behavior. When sending an unlock message to an NSLock object, you must be sure that message is sent from the same thread that sent the initial lock message. Unlocking a lock from a different thread can result in undefined behavior.

可以看到NSLock只是对pthread_mutex_t的一种封装，使其以OC的形式展现给开发者。另外你必须确保在与加锁操作在同一个线程进行解锁。

NSLock实现了NSLocking协议，因此一共提供了四个方法，lock，unlock，trylock以及lockBeforeDate:。

苹果的官方文档给出了两个注意点：

  • 不要用NSLock实现递归锁，一旦在同一线程两次调用lock方法，会造成线程锁死。如果一定要使用递归锁，可以采用NSRecursiveLock。

  • unlock一个未被lock的锁，会导致程序错误。

以上，NSLock适用于一些不太考虑性能的地方，鉴于其封装自pthread_mutex_t，所以性能方面一定略逊于pthread_mutex_t。
